# v7.0.0 Migration Guide

## Breaking Changes

### `AI Search`

#### 1. SearchForAgents metadata removed

**What changed:**
The `SearchForAgents` field and its associated types have been removed from all instance and namespace instance metadata structs.

**Impact:**
Code that reads `SearchForAgents` from responses or sets it in params will no longer compile.

**Removed Types:**
- `InstanceNewResponseMetadataSearchForAgents`
- `InstanceNewParamsMetadataSearchForAgents`
- `InstanceUpdateParamsMetadataSearchForAgents`
- (and all `NamespaceInstance*` equivalents)

**Before (v6.10.0):**
```go
instance, err := client.AISearch.Instances.New(ctx, ai_search.InstanceNewParams{
    AccountID: cloudflare.F("account-id"),
    Metadata: cloudflare.F(ai_search.InstanceNewParamsMetadata{
        CreatedFromAISearchWizard: cloudflare.F(true),
        SearchForAgents: cloudflare.F(ai_search.InstanceNewParamsMetadataSearchForAgents{
            Hostname: cloudflare.F("hostname"),
            ZoneID:   cloudflare.F("zone_id"),
            ZoneName: cloudflare.F("zone_name"),
        }),
        WorkerDomain: cloudflare.F("worker_domain"),
    }),
})
```

**After (v7.0.0):**
```go
instance, err := client.AISearch.Instances.New(ctx, ai_search.InstanceNewParams{
    AccountID: cloudflare.F("account-id"),
    Metadata: cloudflare.F(ai_search.InstanceNewParamsMetadata{
        CreatedFromAISearchWizard: cloudflare.F(true),
        WorkerDomain:              cloudflare.F("worker_domain"),
    }),
})
```

**Actions Needed:**
1. Remove all references to `SearchForAgents` from params and response handling
2. Remove imports of any `*SearchForAgents` types

### `Email Security`

#### 1. Path parameter types changed from `int64` to `string`

**What changed:**
The path parameter for `Delete`, `Edit`, and `Get` methods across five settings sub-resources changed from `int64` to `string`.

**Impact:**
Code passing integer literals or `int64` variables to these methods will no longer compile.

**Affected Services:**
- `AllowPolicies` (`policyID`)
- `BlockSenders` (`patternID`)
- `Domains` (`domainID`)
- `ImpersonationRegistry` (`displayNameID` -> `impersonationRegistryID`)
- `TrustedDomains` (`trustedDomainID`)

**Before (v6.10.0):**
```go
_, err := client.EmailSecurity.Settings.AllowPolicies.Delete(
    ctx,
    int64(2401),
    email_security.SettingAllowPolicyDeleteParams{
        AccountID: cloudflare.F("account_id"),
    },
)
```

**After (v7.0.0):**
```go
_, err := client.EmailSecurity.Settings.AllowPolicies.Delete(
    ctx,
    "2401",
    email_security.SettingAllowPolicyDeleteParams{
        AccountID: cloudflare.F("account_id"),
    },
)
```

**Actions Needed:**
1. Convert all `int64` path parameter arguments to `string` (use `strconv.FormatInt()` or string literals)
2. For `ImpersonationRegistry`, update the parameter name from `displayNameID` to `impersonationRegistryID` in any named-argument usage

---

#### 2. Investigate path parameter renamed

**What changed:**
The `postfixID` parameter in `Investigate.Get`, `Investigate.Move.New`, and `Investigate.Reclassify.New` has been renamed to `investigateID`.

**Impact:**
This is a positional parameter in Go, so existing code will continue to compile. However, any documentation or variable names referencing `postfixID` should be updated for clarity.

**Actions Needed:**
1. Rename local variables from `postfixID` to `investigateID` for consistency (optional)

---

#### 3. Domains BulkDelete method removed

**What changed:**
The `SettingDomainService.BulkDelete` method has been removed from the API.

**Impact:**
Code calling `client.EmailSecurity.Settings.Domains.BulkDelete()` will no longer compile.

**Removed Types:**
- `SettingDomainBulkDeleteResponse`
- `SettingDomainBulkDeleteParams`

**Before (v6.10.0):**
```go
result, err := client.EmailSecurity.Settings.Domains.BulkDelete(ctx,
    email_security.SettingDomainBulkDeleteParams{
        AccountID: cloudflare.F("account_id"),
    },
)
```

**After (v7.0.0):**
```go
// BulkDelete is no longer available.
// Delete domains individually using the Delete method:
_, err := client.EmailSecurity.Settings.Domains.Delete(ctx, "domain-id",
    email_security.SettingDomainDeleteParams{
        AccountID: cloudflare.F("account_id"),
    },
)
```

**Actions Needed:**
1. Replace `BulkDelete` calls with individual `Delete` calls per domain
2. Remove imports of `SettingDomainBulkDeleteParams` and `SettingDomainBulkDeleteResponse`

---

#### 4. TrustedDomains.New return type changed

**What changed:**
`SettingTrustedDomainService.New` returns `*SettingTrustedDomainNewResponse` instead of `*SettingTrustedDomainNewResponseUnion`.

**Impact:**
Code that type-asserts the response to a specific union variant will need updating.

**Before (v6.10.0):**
```go
result, err := client.EmailSecurity.Settings.TrustedDomains.New(ctx, params)
// result is *SettingTrustedDomainNewResponseUnion
```

**After (v7.0.0):**
```go
result, err := client.EmailSecurity.Settings.TrustedDomains.New(ctx, params)
// result is *SettingTrustedDomainNewResponse (concrete type)
```

**Actions Needed:**
1. Replace `SettingTrustedDomainNewResponseUnion` with `SettingTrustedDomainNewResponse`
2. Remove any union type assertions -- the response is now a concrete struct

### `Workers`

#### 1. Observability telemetry filter types restructured

**What changed:**
The filter parameter types for `ObservabilityTelemetry.Keys`, `ObservabilityTelemetry.Query`, and `ObservabilityTelemetry.Values` have been restructured from flat `[]interface{}` arrays to typed discriminated unions supporting nested filter groups.

**Impact:**
Code constructing filter parameters for observability telemetry queries will no longer compile.

**Before (v6.10.0):**
```go
_, err := client.Workers.Observability.Telemetry.Keys(ctx, workers.ObservabilityTelemetryKeysParams{
    AccountID: cloudflare.F("account_id"),
    Filters: cloudflare.F([]interface{}{
        map[string]interface{}{
            "key":       "status",
            "operation": "eq",
            "value":     "200",
        },
    }),
})
```

**After (v7.0.0):**
```go
_, err := client.Workers.Observability.Telemetry.Keys(ctx, workers.ObservabilityTelemetryKeysParams{
    AccountID: cloudflare.F("account_id"),
    Filters: cloudflare.F(workers.ObservabilityTelemetryKeysParamsFiltersObject{
        Filters: cloudflare.F([]workers.ObservabilityTelemetryKeysParamsFiltersObjectFilterUnion{
            workers.ObservabilityTelemetryKeysParamsFiltersObjectFiltersWorkersObservabilityFilterLeaf{
                Key:       cloudflare.F("status"),
                Operation: cloudflare.F(workers.ObservabilityTelemetryKeysParamsFiltersObjectFiltersWorkersObservabilityFilterLeafOperationEq),
                Value:     cloudflare.F[workers.ObservabilityTelemetryKeysParamsFiltersObjectFiltersWorkersObservabilityFilterLeafValueUnion](shared.UnionString("200")),
                Type:      cloudflare.F(workers.ObservabilityTelemetryKeysParamsFiltersObjectFiltersWorkersObservabilityFilterLeafTypeString),
                Kind:      cloudflare.F(workers.ObservabilityTelemetryKeysParamsFiltersObjectFiltersWorkersObservabilityFilterLeafKindFilter),
            },
        }),
    }),
})
```

**Actions Needed:**
1. Replace `[]interface{}` filter construction with typed `FiltersObject` structs
2. Use `FiltersWorkersObservabilityFilterLeaf` for leaf filter conditions
3. Use `FiltersObjectFiltersObject` for nested group filters with `FilterCombination`
4. Set explicit `Type`, `Kind`, and `Operation` fields using the provided enum constants

---

## Summary

1. **AI Search**: Remove `SearchForAgents` from all metadata params and response handling
2. **Email Security**: Convert path parameters from `int64` to `string`; rename `postfixID` to `investigateID`; replace `BulkDelete` with individual `Delete` calls; update `TrustedDomains.New` return type
3. **Workers**: Rebuild observability telemetry filter params using typed discriminated unions
