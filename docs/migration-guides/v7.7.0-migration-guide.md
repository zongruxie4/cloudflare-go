# v7.7.0 Migration Guide

## Breaking Changes

### `email_routing`

#### 1. Rules.List method removed

**What changed:**
The `List` method has been removed from `client.EmailRouting.Rules`. The underlying HTTP endpoint (`GET /zones/{zone_id}/email/routing/rules`) is still supported by the platform but is no longer exposed as a strongly-typed SDK method.

**Impact:**
Code that iterates email routing rules via `client.EmailRouting.Rules.List(...)` will no longer compile.

**Removed Types:**
- `EmailRouting.Rules.List` method
- `RuleListParams`

**Before (v7.6.0):**
```go
iter := client.EmailRouting.Rules.ListAutoPaging(ctx, email_routing.RuleListParams{
    ZoneID: cloudflare.F("zone-id"),
})
for iter.Next() {
    rule := iter.Current()
    // ...
}
```

**After (v7.7.0):**

Either fetch a specific rule by identifier:

```go
rule, err := client.EmailRouting.Rules.Get(ctx, "rule-identifier", email_routing.RuleGetParams{
    ZoneID: cloudflare.F("zone-id"),
})
```

Or call the endpoint directly using the low-level request helper:

```go
var rules []email_routing.EmailRoutingRule
err := client.Get(ctx, "zones/zone-id/email/routing/rules", nil, &rules)
```

**Actions Needed:**
1. Remove all `client.EmailRouting.Rules.List` / `ListAutoPaging` call sites.
2. Replace with per-rule `Get` calls, or issue the HTTP request directly via `client.Get(...)`.

### `ssl`

#### 2. Recommendations.Get method and types removed

**What changed:**
The `Recommendations` sub-resource has been removed from the SSL service. The endpoint `GET /zones/{zone_id}/ssl/recommendation` is no longer exposed by the SDK.

**Impact:**
Code that reads SSL recommendations via `client.SSL.Recommendations.Get(...)` will no longer compile.

**Removed Types:**
- `SSL.Recommendations.Get` method
- `RecommendationGetResponse`
- `RecommendationGetParams`
- `RecommendationService`

**Before (v7.6.0):**
```go
rec, err := client.SSL.Recommendations.Get(ctx, ssl.RecommendationGetParams{
    ZoneID: cloudflare.F("zone-id"),
})
```

**After (v7.7.0):**

Call the endpoint directly using the low-level request helper:

```go
var rec map[string]interface{}
err := client.Get(ctx, "zones/zone-id/ssl/recommendation", nil, &rec)
```

**Actions Needed:**
1. Remove all `client.SSL.Recommendations.Get` call sites and imports of `RecommendationGetResponse` / `RecommendationGetParams`.
2. If your code path still needs the recommendation, call the HTTP endpoint directly via `client.Get(...)`.

### `ai_gateway`, `workflows`, `zero_trust`

#### 3. Union-merged parent fields changed to interface{}

**What changed:**
When the same field name appears on multiple union variants with different Go types, the merged parent struct's field is now typed as `interface{}` (with a `// This field can have the runtime type of ...` comment) instead of the variant-struct type. This matches how the code generator already emits sibling merged fields on the same structs (e.g. `Values`, `Fields`, `AllowedMicrosoftOrganizations`).

**Impact:**
Direct field access on the affected parent fields will no longer compile against the previous type. In practice, direct field access on these fields did not work in v7.6.0 either -- `json.Unmarshal` panicked with `reflect: call of reflect.Value.SetString on struct Value` whenever an API response included the field. Callers that only used `.AsUnion()` for variant handling are unaffected.

**Affected Types:**

| Package | Struct | Field |
| --- | --- | --- |
| `ai_gateway` | `AIGatewayNewResponseSpendLimitsRulesMetadata` | `Mode` |
| `ai_gateway` | `AIGatewayUpdateResponseSpendLimitsRulesMetadata` | `Mode` |
| `ai_gateway` | `AIGatewayListResponseSpendLimitsRulesMetadata` | `Mode` |
| `ai_gateway` | `AIGatewayDeleteResponseSpendLimitsRulesMetadata` | `Mode` |
| `ai_gateway` | `AIGatewayGetResponseSpendLimitsRulesMetadata` | `Mode` |
| `workflows` | `VersionGraphResponseGraphWorkflowPayload` | `Type` |
| `zero_trust` | `DLPEmailAccountMappingNewResponseAuthRequirements` | `Type` |
| `zero_trust` | `DLPEmailAccountMappingGetResponseAuthRequirements` | `Type` |

**Before (v7.6.0):**
```go
// Panics at json.Unmarshal time -- effectively unreachable code:
resp, _ := client.AIGateway.New(ctx, ai_gateway.AIGatewayNewParams{ /* ... */ })
for _, rule := range resp.SpendLimits.Rules {
    for _, meta := range rule.Metadata {
        mode := meta.Mode // AIGatewayNewResponseSpendLimitsRulesMetadataMode (variant struct)
        _ = mode
    }
}
```

**After (v7.7.0):**

Use `.AsUnion()` on the parent to switch on the actual variant:

```go
resp, _ := client.AIGateway.New(ctx, ai_gateway.AIGatewayNewParams{ /* ... */ })
for _, rule := range resp.SpendLimits.Rules {
    for _, meta := range rule.Metadata {
        switch variant := meta.AsUnion().(type) {
        case ai_gateway.AIGatewayNewResponseSpendLimitsRulesMetadataMode:
            // variant.Mode is an AIGatewayNewResponseSpendLimitsRulesMetadataModeMode
            // (e.g. "partition")
        case ai_gateway.AIGatewayNewResponseSpendLimitsRulesMetadataObject:
            // variant.Mode is an AIGatewayNewResponseSpendLimitsRulesMetadataObjectMode
            // (e.g. "filter") and variant.Values is []string
        }
    }
}
```

If you must read the raw value without a type switch, type-assert on the runtime type documented in the field comment:

```go
if s, ok := meta.Mode.(ai_gateway.AIGatewayNewResponseSpendLimitsRulesMetadataModeMode); ok {
    // ...
}
```

**Actions Needed:**
1. Replace direct field access (`x.Mode`, `x.Type`) on the affected structs with `x.AsUnion()` and a type switch on the concrete variant.
2. If a type switch is not practical, type-assert on one of the concrete runtime types listed in the field comment.
3. Anywhere code was calling into the field expecting a struct, delete the call: it was unreachable in v7.6.0 due to the unmarshal panic.

---

## Summary of Breaking Changes

1. `email_routing.Rules.List` method removed (and `RuleListParams`).
2. `ssl.Recommendations.Get` method removed along with `RecommendationGetResponse`, `RecommendationGetParams`, and `RecommendationService`.
3. Eight union-merged parent-struct fields changed from a variant-struct type to `interface{}` across `ai_gateway`, `workflows`, and `zero_trust`.
