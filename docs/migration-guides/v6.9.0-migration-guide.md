# v6.9.0 Migration Guide

## Breaking Changes

### `AI Gateway`

#### 1. AccountID, AccountTag, and InternalID fields removed from response types

**What changed:**
The `AccountID`, `AccountTag`, and `InternalID` fields have been removed from all AI Gateway response types.

**Impact:**
Code that accesses `AccountID`, `AccountTag`, or `InternalID` fields on AI Gateway responses will no longer compile.

**Affected Types:**
- `AIGatewayNewResponse`
- `AIGatewayUpdateResponse`
- `AIGatewayListResponse`
- `AIGatewayDeleteResponse`
- `AIGatewayGetResponse`
- `DynamicRoutingNewResponse`
- `DynamicRoutingDeleteResponse`
- `DynamicRoutingNewDeploymentResponse`
- `DynamicRoutingNewVersionResponse`
- `DynamicRoutingGetResponse`
- `DynamicRoutingGetVersionResponse`

**Before (v6.8.0):**
```go
import (
    cloudflare "github.com/cloudflare/cloudflare-go/v7"
    "github.com/cloudflare/cloudflare-go/v7/ai_gateway"
    "github.com/cloudflare/cloudflare-go/v7/option"
)

client, _ := cloudflare.NewClient(option.WithAPIToken("..."))

gateway, err := client.AI.Gateways.New(
    context.TODO(),
    ai_gateway.AIGatewayNewParams{
        AccountID: cloudflare.F("account-id"),
    },
)

// Accessing removed fields
accountID := gateway.AccountID
accountTag := gateway.AccountTag
internalID := gateway.InternalID
```

**After (v6.9.0):**
```go
import (
    cloudflare "github.com/cloudflare/cloudflare-go/v7"
    "github.com/cloudflare/cloudflare-go/v7/ai_gateway"
    "github.com/cloudflare/cloudflare-go/v7/option"
)

client, _ := cloudflare.NewClient(option.WithAPIToken("..."))

gateway, err := client.AI.Gateways.New(
    context.TODO(),
    ai_gateway.AIGatewayNewParams{
        AccountID: cloudflare.F("account-id"),
    },
)

// AccountID, AccountTag, and InternalID have been removed from responses
// Use the ID field instead
gatewayID := gateway.ID
```

**Actions Needed:**
1. Remove all accesses to `.AccountID`, `.AccountTag`, and `.InternalID` fields on AI Gateway response types
2. If you need the account ID, store it separately before making the API call

### `AI Search`

#### 1. KeywordMatchMode enum values changed

**What changed:**
The `KeywordMatchMode` enum values have been renamed from `exact_match`/`fuzzy_match` to `and`/`or`.

| Before | After |
|--------|-------|
| `InstanceNewParamsRetrievalOptionsKeywordMatchModeExactMatch` | `InstanceNewParamsRetrievalOptionsKeywordMatchModeAnd` |
| `InstanceNewParamsRetrievalOptionsKeywordMatchModeFuzzyMatch` | `InstanceNewParamsRetrievalOptionsKeywordMatchModeOr` |
| `InstanceUpdateParamsRetrievalOptionsKeywordMatchModeExactMatch` | `InstanceUpdateParamsRetrievalOptionsKeywordMatchModeAnd` |
| `InstanceUpdateParamsRetrievalOptionsKeywordMatchModeFuzzyMatch` | `InstanceUpdateParamsRetrievalOptionsKeywordMatchModeOr` |

**Impact:**
Code using the old enum constants will no longer compile.

**Before (v6.8.0):**
```go
import (
    cloudflare "github.com/cloudflare/cloudflare-go/v7"
    "github.com/cloudflare/cloudflare-go/v7/ai_search"
    "github.com/cloudflare/cloudflare-go/v7/option"
)

client, _ := cloudflare.NewClient(option.WithAPIToken("..."))

instance, err := client.AISearch.Instances.New(
    context.TODO(),
    ai_search.InstanceNewParams{
        AccountID: cloudflare.F("account-id"),
        RetrievalOptions: ai_search.F(ai_search.InstanceNewParamsRetrievalOptions{
            KeywordMatchMode: ai_search.F(ai_search.InstanceNewParamsRetrievalOptionsKeywordMatchModeExactMatch),
        }),
    },
)
```

**After (v6.9.0):**
```go
import (
    cloudflare "github.com/cloudflare/cloudflare-go/v7"
    "github.com/cloudflare/cloudflare-go/v7/ai_search"
    "github.com/cloudflare/cloudflare-go/v7/option"
)

client, _ := cloudflare.NewClient(option.WithAPIToken("..."))

instance, err := client.AISearch.Instances.New(
    context.TODO(),
    ai_search.InstanceNewParams{
        AccountID: cloudflare.F("account-id"),
        RetrievalOptions: ai_search.F(ai_search.InstanceNewParamsRetrievalOptions{
            KeywordMatchMode: ai_search.F(ai_search.InstanceNewParamsRetrievalOptionsKeywordMatchModeAnd),
        }),
    },
)
```

**Actions Needed:**
1. Replace all occurrences of `KeywordMatchModeExactMatch` with `KeywordMatchModeAnd`
2. Replace all occurrences of `KeywordMatchModeFuzzyMatch` with `KeywordMatchModeOr`

**Note:** The API itself accepts both old and new values for backward compatibility, but the SDK types use the new names.

### `Custom Hostnames`

#### 1. Hostname parameter type changed

**What changed:**
The `Hostname` field in `CustomHostnameListParams` changed from a simple string to a structured object with a `Contain` field.

| Field | Before | After |
|-------|--------|-------|
| `Hostname` | `param.Field[string]` | `param.Field[CustomHostnameListParamsHostname]` |

**Impact:**
Code that passes a string value directly to `Hostname` will no longer compile.

**Before (v6.8.0):**
```go
import (
    cloudflare "github.com/cloudflare/cloudflare-go/v7"
    "github.com/cloudflare/cloudflare-go/v7/custom_hostnames"
    "github.com/cloudflare/cloudflare-go/v7/option"
)

client, _ := cloudflare.NewClient(option.WithAPIToken("..."))

hostnames, err := client.CustomHostnames.List(
    context.TODO(),
    custom_hostnames.CustomHostnameListParams{
        ZoneID:   cloudflare.F("zone-id"),
        Hostname: cloudflare.F("example.com"),
    },
)
```

**After (v6.9.0):**
```go
import (
    cloudflare "github.com/cloudflare/cloudflare-go/v7"
    "github.com/cloudflare/cloudflare-go/v7/custom_hostnames"
    "github.com/cloudflare/cloudflare-go/v7/option"
)

client, _ := cloudflare.NewClient(option.WithAPIToken("..."))

hostnames, err := client.CustomHostnames.List(
    context.TODO(),
    custom_hostnames.CustomHostnameListParams{
        ZoneID: cloudflare.F("zone-id"),
        Hostname: cloudflare.F(custom_hostnames.CustomHostnameListParamsHostname{
            Contain: cloudflare.F("example.com"),
        }),
    },
)
```

**Actions Needed:**
1. Wrap all `Hostname` values in `custom_hostnames.CustomHostnameListParamsHostname{Contain: cloudflare.F("...")}`
2. Review if you need the new filter parameters that were also added (`CertificateAuthority`, `HostnameStatus`, `SSLStatus`, `Wildcard`)

### `Connectivity Directory Services`

#### 1. ServiceConfig changed to discriminated union

**What changed:**
The `DirectoryServiceNewParams` and `DirectoryServiceUpdateParams` structures have been refactored. The flat structure with `Host`, `Name`, `Type`, `HTTPPort`, `HTTPSPort` fields has been replaced with a discriminated union based on the `type` field.

**Impact:**
Code using the old flat parameter structure will no longer compile.

**Before (v6.8.0):**
```go
import (
    cloudflare "github.com/cloudflare/cloudflare-go/v7"
    "github.com/cloudflare/cloudflare-go/v7/connectivity"
    "github.com/cloudflare/cloudflare-go/v7/option"
)

client, _ := cloudflare.NewClient(option.WithAPIToken("..."))

service, err := client.Connectivity.Directory.Services.New(
    context.TODO(),
    connectivity.DirectoryServiceNewParams{
        AccountID: cloudflare.F("account-id"),
        Host: cloudflare.F[connectivity.DirectoryServiceNewParamsHostUnion](
            connectivity.DirectoryServiceNewParamsHostInfraHostnameHost{
                Hostname: cloudflare.F("api.example.com"),
                ResolverNetwork: cloudflare.F(connectivity.DirectoryServiceNewParamsHostInfraHostnameHostResolverNetwork{
                    TunnelID:    cloudflare.F("tunnel-id"),
                    ResolverIPs: cloudflare.F([]string{"10.0.0.1"}),
                }),
            },
        ),
        Name:      cloudflare.F("web-server"),
        Type:      cloudflare.F(connectivity.DirectoryServiceNewParamsTypeHTTP),
        HTTPPort:  cloudflare.F(int64(8080)),
        HTTPSPort: cloudflare.F(int64(8443)),
    },
)
```

**After (v6.9.0):**
```go
import (
    cloudflare "github.com/cloudflare/cloudflare-go/v7"
    "github.com/cloudflare/cloudflare-go/v7/connectivity"
    "github.com/cloudflare/cloudflare-go/v7/option"
)

client, _ := cloudflare.NewClient(option.WithAPIToken("..."))

// The parameters are now structured differently based on service type
// The exact structure depends on whether you're creating HTTP or TCP services
// The SDK now uses discriminated union types

// For HTTP services:
service, err := client.Connectivity.Directory.Services.New(
    context.TODO(),
    connectivity.DirectoryServiceNewParams{
        AccountID: cloudflare.F("account-id"),
        // Parameters now vary based on the service type
        // Check the updated SDK documentation for the exact structure
    },
)
```

**Actions Needed:**
1. Review the updated `connectivity.DirectoryServiceNewParams` and `connectivity.DirectoryServiceUpdateParams` structures
2. Update your code to use the new discriminated union parameter types
3. Note that TCP services are now also supported alongside HTTP services

### `Billing`

#### 1. PayGo endpoint parameter changes

**What changed:**
The new `billing.Usage.Paygo()` endpoint uses `From` and `To` date parameters instead of the previously available `LastMonthPeriodStart` and `LastYearPeriodStart`.

**Impact:**
If you were using the PayGo endpoint in beta, update your parameter usage.

**Before (beta):**
```go
// Old parameters that were removed
params := billing.UsagePaygoParams{
    AccountID:            cloudflare.F("account-id"),
    LastMonthPeriodStart: cloudflare.F(int64(202501)),
    LastYearPeriodStart:  cloudflare.F(int64(2025)),
}
```

**After (v6.9.0):**
```go
params := billing.UsagePaygoParams{
    AccountID: cloudflare.F("account-id"),
    From:      cloudflare.F(time.Date(2025, 1, 1, 0, 0, 0, 0, time.UTC)),
    To:        cloudflare.F(time.Date(2025, 1, 31, 0, 0, 0, 0, time.UTC)),
}
```

**Actions Needed:**
1. Replace `LastMonthPeriodStart` and `LastYearPeriodStart` with `From` and `To` date parameters
2. Use proper `time.Time` values instead of integer period identifiers

### `AI Search - InstanceItem`

#### 1. InstanceItem service methods removed

**What changed:**
The `InstanceItem` service methods have been completely removed. The service structure still exists at `client.AISearch.Instances.Items` but no longer provides any methods.

**Removed Methods:**
- `List()` - List indexed items in an AI Search instance
- `ListAutoPaging()` - Auto-paging variant for listing items
- `Get()` - Get a specific indexed item by ID

**Removed Types:**
- `InstanceItemListResponse`
- `InstanceItemGetResponse`
- `InstanceItemListParams`
- `InstanceItemGetParams`
- Associated enum types (`InstanceItemListResponseStatus`, `InstanceItemListResponseNextAction`, etc.)

**Impact:**
Code using any InstanceItem methods will no longer compile.

**Before (v6.8.0):**
```go
import (
    cloudflare "github.com/cloudflare/cloudflare-go/v7"
    "github.com/cloudflare/cloudflare-go/v7/option"
)

client, _ := cloudflare.NewClient(option.WithAPIToken("..."))

// List items in an AI Search instance
items, err := client.AISearch.Instances.Items.List(
    context.TODO(),
    "instance-id",
    ai_search.InstanceItemListParams{
        AccountID: cloudflare.F("account-id"),
    },
)

// Get a specific item
item, err := client.AISearch.Instances.Items.Get(
    context.TODO(),
    "instance-id",
    "item-id",
    ai_search.InstanceItemGetParams{
        AccountID: cloudflare.F("account-id"),
    },
)
```

**After (v6.9.0):**
```go
// The InstanceItem service methods have been removed.
// There is currently no replacement for listing or getting indexed items.
// Contact Cloudflare support if you need this functionality.
```

**Actions Needed:**
1. Remove all code that uses `client.AISearch.Instances.Items.List()` or `client.AISearch.Instances.Items.Get()`
2. If you need this functionality, contact Cloudflare support

### `Workers`

#### 1. Domain service return type changes

**What changed:**
The `workers.Domain` service methods now return specific response types instead of the generic `Domain` type. Additionally, the `Delete()` method now returns a response object.

**Impact:**
Code that uses the `Domain` type directly will need to update to the new specific response types.

**Before (v6.8.0):**
```go
import (
    cloudflare "github.com/cloudflare/cloudflare-go/v7"
    "github.com/cloudflare/cloudflare-go/v7/workers"
    "github.com/cloudflare/cloudflare-go/v7/option"
)

client, _ := cloudflare.NewClient(option.WithAPIToken("..."))

// Update returned *Domain
domain, err := client.Workers.Domains.Update(ctx, workers.DomainUpdateParams{...})
var d *workers.Domain = domain

// List returned SinglePage[Domain]
domains, err := client.Workers.Domains.List(ctx, workers.DomainListParams{...})
for _, d := range domains.Result {
    var item workers.Domain = d
}

// Delete returned only error
err = client.Workers.Domains.Delete(ctx, "domain-id", workers.DomainDeleteParams{...})

// Get returned *Domain
domain, err := client.Workers.Domains.Get(ctx, "domain-id", workers.DomainGetParams{...})
var d *workers.Domain = domain
```

**After (v6.9.0):**
```go
import (
    cloudflare "github.com/cloudflare/cloudflare-go/v7"
    "github.com/cloudflare/cloudflare-go/v7/workers"
    "github.com/cloudflare/cloudflare-go/v7/option"
)

client, _ := cloudflare.NewClient(option.WithAPIToken("..."))

// Update now returns *DomainUpdateResponse
domain, err := client.Workers.Domains.Update(ctx, workers.DomainUpdateParams{...})
var d *workers.DomainUpdateResponse = domain

// List now returns SinglePage[DomainListResponse]
domains, err := client.Workers.Domains.List(ctx, workers.DomainListParams{...})
for _, d := range domains.Result {
    var item workers.DomainListResponse = d
}

// Delete now returns (*DomainDeleteResponse, error)
resp, err := client.Workers.Domains.Delete(ctx, "domain-id", workers.DomainDeleteParams{...})

// Get now returns *DomainGetResponse
domain, err := client.Workers.Domains.Get(ctx, "domain-id", workers.DomainGetParams{...})
var d *workers.DomainGetResponse = domain
```

**Actions Needed:**
1. Update type declarations from `*workers.Domain` to the specific response types
2. Update `Delete()` calls to handle the new return value `(*DomainDeleteResponse, error)`

### `Zero Trust`

#### 1. NetworkSubnet response type consolidation

**What changed:**
The `NetworkSubnet` service methods now return the shared `Subnet` type instead of endpoint-specific response types. The following types have been removed:

- `NetworkSubnetListResponse`
- `NetworkSubnetWARPNewResponse`
- `NetworkSubnetWARPEditResponse`
- `NetworkSubnetWARPGetResponse`
- `NetworkSubnetCloudflareSourceUpdateResponse`

**Impact:**
Code that references the removed types will no longer compile.

**Before (v6.8.0):**
```go
import (
    cloudflare "github.com/cloudflare/cloudflare-go/v7"
    "github.com/cloudflare/cloudflare-go/v7/zero_trust"
    "github.com/cloudflare/cloudflare-go/v7/option"
)

client, _ := cloudflare.NewClient(option.WithAPIToken("..."))

// List returned V4PagePaginationArray[NetworkSubnetListResponse]
subnets, err := client.ZeroTrust.Networks.Subnets.List(ctx, zero_trust.NetworkSubnetListParams{...})
for _, subnet := range subnets.Result {
    var s zero_trust.NetworkSubnetListResponse = subnet
}

// WARP New returned *NetworkSubnetWARPNewResponse
subnet, err := client.ZeroTrust.Networks.Subnets.WARP.New(ctx, zero_trust.NetworkSubnetWARPNewParams{...})
var s *zero_trust.NetworkSubnetWARPNewResponse = subnet

// WARP Edit returned *NetworkSubnetWARPEditResponse
subnet, err := client.ZeroTrust.Networks.Subnets.WARP.Edit(ctx, "subnet-id", zero_trust.NetworkSubnetWARPEditParams{...})
var s *zero_trust.NetworkSubnetWARPEditResponse = subnet

// WARP Get returned *NetworkSubnetWARPGetResponse
subnet, err := client.ZeroTrust.Networks.Subnets.WARP.Get(ctx, "subnet-id", zero_trust.NetworkSubnetWARPGetParams{...})
var s *zero_trust.NetworkSubnetWARPGetResponse = subnet

// CloudflareSource Update returned *NetworkSubnetCloudflareSourceUpdateResponse
subnet, err := client.ZeroTrust.Networks.Subnets.CloudflareSource.Update(ctx, "ipv4", zero_trust.NetworkSubnetCloudflareSourceUpdateParams{...})
var s *zero_trust.NetworkSubnetCloudflareSourceUpdateResponse = subnet
```

**After (v6.9.0):**
```go
import (
    cloudflare "github.com/cloudflare/cloudflare-go/v7"
    "github.com/cloudflare/cloudflare-go/v7/zero_trust"
    "github.com/cloudflare/cloudflare-go/v7/option"
)

client, _ := cloudflare.NewClient(option.WithAPIToken("..."))

// List now returns V4PagePaginationArray[Subnet]
subnets, err := client.ZeroTrust.Networks.Subnets.List(ctx, zero_trust.NetworkSubnetListParams{...})
for _, subnet := range subnets.Result {
    var s zero_trust.Subnet = subnet
}

// WARP New now returns *Subnet
subnet, err := client.ZeroTrust.Networks.Subnets.WARP.New(ctx, zero_trust.NetworkSubnetWARPNewParams{...})
var s *zero_trust.Subnet = subnet

// WARP Edit now returns *Subnet
subnet, err := client.ZeroTrust.Networks.Subnets.WARP.Edit(ctx, "subnet-id", zero_trust.NetworkSubnetWARPEditParams{...})

// WARP Get now returns *Subnet

```

**Actions Needed:**
1. Replace all references to `NetworkSubnetListResponse` with `Subnet`
2. Replace all references to `NetworkSubnetWARPNewResponse` with `Subnet`
3. Replace all references to `NetworkSubnetWARPEditResponse` with `Subnet`
4. Replace all references to `NetworkSubnetWARPGetResponse` with `Subnet`
5. Replace all references to `NetworkSubnetCloudflareSourceUpdateResponse` with `Subnet`

## Summary

The v6.9.0 release includes several breaking changes across multiple services:

1. **AI Gateway** - Removed `AccountID`, `AccountTag`, and `InternalID` from all response types
2. **AI Search** - Renamed enum values for `KeywordMatchMode`
3. **AI Search** - Removed `VectorizeName` field from instance response types
4. **AI Search - InstanceItem** - Removed all service methods (List, Get, etc.)
5. **Custom Hostnames** - Changed `Hostname` parameter from string to structured object
6. **Connectivity** - Refactored service configuration to discriminated union types
7. **Billing** - Updated PayGo endpoint parameters
8. **Workers** - Domain service return types changed from `Domain` to specific response types
9. **Zero Trust** - NetworkSubnet response types consolidated to shared `Subnet` type
10. **Zero Trust** - Removed `MfaBypass` field from all MFA configuration types
11. **Zero Trust** - Removed `MfaConfigurationAllowed` field from Organization types

Please review all changes above and update your code accordingly before upgrading to v6.9.0.
